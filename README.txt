TRABALHO DE ESTRUTURA DE DADOSGQ2Os 4 pontos que compõem a nota de GQ2 de Estrutura de Dados serão contabilizados da seguinte forma:Condições necessárias/fundamentais para a pontuação - individual:1.O aluno precisa ter um login no GitLab (www.gitlab.com);2.O aluno deve estar alocado em uma equipe de até 5 pessoas;3.O aluno deve ter participação efetiva no desenvolvimento dos códigos (O ideal, é que o líder de cada equipe divida as funções de cada exercício para os membros da equipe.);Condições de pontuação para o grupo:1.Atendendo as condições fundamentais, suprarelacionadas. O aluno será pontuado da seguinte forma:2.Os códigos devem ser desenvolvido usando o SO LINUX ou similar. O windows não será aceito. Caso contrário, a atividade e todos os membros da equipe receberão ZERO.3.O código precisa ser submetido, TAMBÉM, via ferramenta acadêmica também (AS DATAS LIMITE DEVEM SER RESPEITADAS);4.Cada código deve ser único por equipe, evitando cópias dos trabalhos das demais equipes (Em caso de cópia, os trabalhos das equipes envolvidas serão ZERADOS);5.Os códigos sem indentação serão ZERADOS;6.Todos os códigos devem ser desenvolvidos utilizando ponteiros;7.Os códigos devem ter soluções completas. Códigos parciais e/ou incompletos serão totalmente desconsiderados.
8.Cada exercício deve ter a sua subpasta definida com o nome AtividadeX, em que X é a sequência correta das atividades solicitadas em sala:
	<Diretório principal>
		<Atividade 1>
		<Atividade 2>
		<Atividade 3>
		<Atividade 4>
9. LEMBREM-SE: O TRABALHO É EM GRUPO, MAS A PONTUAÇÃO É INDIVIDUAL. VOU ANALISAR OS RELATÓRIOS DE CONTRIBUIÇÃO POR USUÁRIO.
